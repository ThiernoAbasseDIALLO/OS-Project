\doxysubsection{Description détaillée}
Logique de l\textquotesingle{}algorithme d\textquotesingle{}ordonnancement SJF non-\/préemptif. 

Stratégie \+: parmi tous les processus prêts, on exécute celui dont le burst CPU courant est le plus court. Non-\/préemptif. Les E/S sont parallélisées.

La variable courant est persistante \+: une fois sélectionné, le processus garde le CPU jusqu\textquotesingle{}à la fin de son burst, même si un processus plus court arrive entre-\/temps. C\textquotesingle{}est ce qui différencie SJF (non-\/préemptif) de SJRF (préemptif). Pas d\textquotesingle{}affichage ni d\textquotesingle{}export CSV — géré dans \mbox{\hyperlink{main_8c}{main.\+c}}.

Participation \+: 33\% pour les 3 auteurs (DIALLO, DOSSO, MAREGA).
